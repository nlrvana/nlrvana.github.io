1. 根据流量包分析首个进行扫描攻击的IP是

被攻击的主机 ip 是192.168.0.211
这里过滤一下
ip.dst == 192.168.0.211
这是正常的ping包，继续往下看
疑似存在端口探测行为，这里继续进行过滤一下
ip.dst == 192.168.0.211 && ip.src== 192.168.0.223
可以确定为端口探测+目录扫描
攻击ip是192.168.0.223
flag{192.168.0.223}

2. 根据流量包分析第二个扫描攻击的IP和漏扫工具，以flag{x.x.x.x&工具名}

ip.dst == 192.168.0.211 && ip.src != 192.168.0.223 && http
其中192.168.0.200一直在做漏洞扫描

查找一下漏洞扫描器特征，从UA、字典特征，行为特征、DNSlog地址、url中的某些地址查看
其中 url 地址中部分为bxss.me
这是awvs(acunetix)扫描器的特征
flag{192.168.0.200&acunetix}

3、提交频繁爆破密钥的IP及爆破次数，以flag{ip&次数}提交

服务器内存在ruoyi框架，存在shiro，这里爆破的是shiro密钥
这里一直在/login路由爆破
过滤一下
ip.dst == 192.168.0.211 && ip.src == 192.168.0.226 &&http.request.uri=="/login"
去除开头三次
一共爆破了1068次
flag{192.168.0.226&1068}

4、提交攻击者利用成功的密钥，以flag{xxxxx}提交

ip.dst == 192.168.0.211 && ip.src != 192.168.0.223 && http.request.uri contains login
其中242在226之后，并且一开始就在爆破密钥，这里过滤一下
ip.dst == 192.168.0.211&&ip.src==192.168.0.242
这里并没有爆破成功，而是下载了/actuator/heapdump
这里下载下来，进行分析
flag{c+3hFGPjbgzGdrC+MHgoRQ==}

5、提交攻击者获取到的所有服务的弱口令，多个以&提交

分析heapdump即可，得到
flag{ruoyi123&admin123&123456}

6、根据应急响应方法，提交利用漏洞成功的端口，多个以&连接

根据上面的分析，得知漏洞利用成功端口是
分别是泄露了heapdump和ruoyi框架漏洞利用
flag{9988&12333}

7、根据流量包分析，提交攻击者利用密钥探测成功的dnslog地址

筛选 DNS 协议和主机
dns&&ip.src==192.168.0.211
flag{1dvrle.dnslog.cn}

8、根据流量包分析，提交攻击者反弹shell的地址和端口

漏洞利用是在ruoyi进行的，直接过滤http流量即可
ip.dst==192.168.0.211&&ip.src==192.168.0.242&&http
从heapdump往后看即可
直接找到最后一个
flag{192.168.0.251:8888}

9、攻击者在主机放置了fscan(已改名)，经扫描拿下一台永恒之蓝漏洞主机，以此为线索进行提交fscan绝对路径

因为是反弹 shell，在TCP/IP协议中，会话需要经过三次握手和四次挥手，SYN，ACK包中存在明文流量
ip.dst==192.168.0.211&&ip.src==192.168.0.251&&tcp.port==8888 &&tcp.flags.syn == 1 && tcp.flags.ack == 1
flag{/opt/.f/.s/.c/.a/.n}

10、另类方法：提交此fscan工具的MD5值

flag{b8053bcd04ce9d7d19c7f36830a9f26b}

11、攻击者为了权限维持，在主机放置了仿真远控工具，需提交此远控工具的下载地址

权限位置，一般是定时任务
cat /etc/cron* /etc/at* /etc/anacrontab /var/spool/cron/crontabs/root 2>/dev/null | grep -v "^#"
每隔十分钟执行一次.qxwc.sh
flag{http://zhoudinb.com:12345/qxwc.sh}

12、攻击者就知道你会这样找到，所以又创建了一条相关的脚本，使用其他方法进行下载，提交脚本的绝对路径

Linux 中还可以通过开机自启动，来做权限维持，通过systemctl查看
/etc/systemd/system/创建任务是通过此目录下进行定位文件名创建任务名
happy.service服务可疑，打开查看
flag{/home/security/upload/.CCC/.happy.sh}

13、攻击者创建了一个隐藏用户，提交此用户的用户名

在/root/.ssh/id_rsa.pub下，发现可疑用户
flag{xj1zhoudi@kali}
